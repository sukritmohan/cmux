cmux Mobile Companion -- Navigation Models

Three approaches for mapping the desktop hierarchy (Windows > Workspaces > Panes > Surfaces) onto a phone screen. Each approach solves the core tension differently: how do you give fast access to many surfaces across many workspaces on a 6-inch display?

Approach A
Surface Strip + Bottom Nav
Horizontal scrolling chip strip shows all surfaces for the current workspace. Bottom nav switches between workspace list, terminal, browser, and files.
Approach B
Card Stack + Swipe Nav
Each surface is a full-screen card. Swipe left/right to move between surfaces. Pull down for workspace drawer. Minimal persistent chrome.
Approach C
Unified Feed + Quick Switcher
All workspace activity in a single scrollable feed. Double-tap home to invoke a quick switcher overlay that jumps to any surface.
Approach A Recommended
Surface Strip + Bottom Nav
Core insight: Flatten the Panes > Surfaces hierarchy into a single horizontal strip of chips grouped by pane. The strip acts as a "linearized split view" -- the same surfaces, just accessed sequentially instead of spatially. Bottom navigation handles the top-level mode switching (workspaces, terminal view, browser, files, shell).
Workspace Nav
Bottom tab "Workspaces" opens the full sidebar list mirroring desktop. Tap a workspace to enter it. Current workspace name persists in the header.
Surface Access
Horizontal chip strip below the header shows all surfaces grouped by pane with subtle dividers. Tap to switch. Pane labels (Left, Right, Bottom) provide spatial context.
View Switching
Bottom nav: Workspaces | Terminal | Browser | Files | Shell. Terminal and Browser views share the surface strip; Files and Shell are standalone.
Notifications
Toast banners slide in from top with workspace + surface context. Notification dots appear on workspace list items and on surface chips. Bell icon in header shows history.
Strengths
  • All surfaces visible and one-tap reachable
  • Pane grouping preserves desktop mental model
  • Familiar bottom-nav pattern (zero learning curve)
  • Notification dots on chips give at-a-glance status
  • Scales well from 2 to 12+ surfaces
Tradeoffs
  • Two scrolling axes (strip horizontal, content vertical)
  • Chip strip consumes ~40px of vertical space
  • Many surfaces may require scrolling the strip

Approach B
Card Stack + Swipe Navigation
Core insight: Treat each surface as a full-screen card. Swipe left/right to move between surfaces within a workspace (like swiping between pages). Pull down from the top to reveal a workspace drawer. No persistent bottom nav -- maximum screen real estate for terminal content. A floating action button provides quick access to browser, files, and shell.
Workspace Nav
Pull down gesture reveals a half-sheet drawer with workspace list. Current workspace shown as a compact pill at top of screen. Long-press the pill for quick workspace switching.
Surface Access
Horizontal swipe between surfaces. Page dots at bottom indicate position. Pane boundaries shown as a thicker dot gap. Feels like swiping between app pages on a home screen.
View Switching
Floating action button (bottom-right) expands into radial menu: Browser, Files, Shell. Terminal surfaces are the default card content -- no mode switching needed.
Notifications
Notification badge on workspace pill. Full toast banners. Tap toast to jump directly to the relevant workspace and surface.
Strengths
  • Maximum terminal real estate (no bottom nav or strip)
  • Gesture-first feels native and fluid
  • Immersive experience for focused terminal work
  • Elegant minimal chrome
Tradeoffs
  • Swipe navigation is not discoverable for new users
  • Can not jump to a specific surface -- must swipe through
  • Pull-down gesture may conflict with system notifications
  • Harder to maintain spatial awareness of all surfaces
  • Gesture vocabulary requires learning

Approach C
Unified Feed + Quick Switcher
Core insight: Instead of mirroring the desktop hierarchy, reimagine it as a live activity feed. All workspaces stream their status into a single timeline -- agent completions, build results, server logs, git pushes. Tapping any feed item opens that surface full-screen. Double-tap the bottom bar to summon a quick switcher (like Cmd+K) that fuzzy-searches across all workspaces and surfaces.
Workspace Nav
No dedicated workspace list. The feed implicitly groups by workspace. Filter chips at top let you scope to a specific workspace. Quick switcher also filters by workspace.
Surface Access
Tap any feed card to open that surface full-screen. Quick switcher (double-tap home) shows all surfaces with fuzzy search. Recent surfaces shown first.
View Switching
Bottom nav: Feed | Terminal | Browser | Files. Terminal tab remembers your last-viewed surface. Feed is the home base you always return to.
Notifications
Notifications ARE the feed. New events appear at top with highlight animation. No separate notification system needed -- the feed is the notification center.
Strengths
  • Perfectly suited to "monitoring" use case on mobile
  • No hierarchy mapping needed -- everything is flat
  • Quick switcher is extremely fast for power users
  • Notifications integrated naturally into the feed
  • Novel approach that plays to mobile strengths
Tradeoffs
  • Loses spatial relationship between surfaces
  • Feed can get noisy with many active workspaces
  • Quick switcher requires learning the double-tap gesture
  • Less useful for active terminal work, more for monitoring
  • Mental model diverges significantly from desktop

Recommendation
Approach A (Surface Strip + Bottom Nav) is the strongest foundation. It preserves the desktop mental model (pane grouping via the strip), provides instant one-tap access to any surface, uses a familiar navigation pattern, and works equally well for active terminal work and passive monitoring. The 40px vertical cost of the surface strip is a worthwhile trade for the discoverability and speed it provides.

However, consider cherry-picking the quick switcher from Approach C as a power-user shortcut layered on top. Double-tap the bottom bar or use a keyboard shortcut (for users with Bluetooth keyboards) to invoke fuzzy search across all workspaces and surfaces. This gives you the best of both worlds: an immediately understandable interface for casual use, and a lightning-fast escape hatch for power users managing 10+ workspaces.
