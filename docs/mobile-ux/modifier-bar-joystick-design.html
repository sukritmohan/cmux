Modifier Bar Redesign

Gesture Joystick + Modifier Keys

Inspired by Echo SSH — a circular crosshair joystick replaces the tiny arrow cluster. Swipe for a single arrow. Hold and drag for rapid repeat. The modifier bar gains esc, shift, ctrl, opt, cmd, and a signature amber Return.

02 — Joystick Interaction

How the Joystick Works

Two modes from one button: quick swipe for single arrow keys, press-and-hold with drag for continuous repeat with accelerating speed.

✥
Resting
Circular crosshair button sits in the bar. Subtle border gives it a physical, pressable look.
✥ →
Swipe → Single Arrow
Quick flick from the button in any direction. Fires one arrow key. Min 8px distance. ⫸ light tick
✥ →
Hold + Drag → Repeat
Press down >200ms, then drag past 12px threshold. Arrow repeats 200ms→40ms. ⫸ continuous ticks
✥
Release
Lift thumb. Repeat stops instantly. Button springs back with 0.2s ease-out-expo.

04 — Behavior Specification

Gesture Reference

Complete interaction table for the joystick button. All gestures, thresholds, and haptic feedback.

GestureBehaviorFeedback
Swipe from button Quick flick originating from the joystick in any cardinal direction (≥8px). Fires one arrow key. Touch can end anywhere. 90° cardinal sectors, diagonal snaps to nearest axis. HapticFeedback.lightImpact(). Button briefly flashes amber.
Press + hold Finger down for >200ms without movement (<4px). Enters drag-ready state. No arrow fires yet. Button contracts to 92% scale. Crosshair glows amber. HapticFeedback.mediumImpact().
Hold + drag While held, drag past 12px threshold in any cardinal direction. First arrow fires immediately. Key repeats while held past threshold. Repeat rate: 200ms → 40ms (over ~1s). HapticFeedback.selectionClick() on each repeat.
Change direction While held, drag back through center and out in a new direction. Old direction stops, new direction starts repeating immediately. HapticFeedback.selectionClick() on center crossing. New direction lights up.
Release Lift finger. Repeat stops immediately. Button returns to resting state. Spring-back animation (0.2s ease-out-expo).
Modifier + swipe Tap a modifier (e.g. ctrl) to sticky it, then swipe joystick. Sends modified arrow (e.g. Ctrl+→ = \x1b[1;5C). Modifier auto-releases after. Active modifier stays amber until consumed by the arrow event.
05 — Escape Sequences

Terminal Protocol (xterm)

Arrow keys follow xterm standard CSI sequences. Modifier parameter follows 1;{code} convention.

06 — Design Rationale

Why These Choices

Joystick over Buttons

The old 26×14px inverted-T was unusable on mobile. Instead of making bigger buttons (which eat bar space), a single 40×40px circular joystick handles all four directions through gesture. The entire button is your touch target.

Echo SSH Inspiration

Echo's crosshair button (✥) proves that gesture-based arrow input works on touchscreens. We adapt their visual language while adding hold+drag for repeat, which Echo handles differently via their toolbar.

Modifier Capsule

⇧ ctrl ⌥ ⌘ grouped in a connected capsule reads as one unit. Tap to sticky-toggle (amber = active), double-tap to lock (amber + underline). Auto-releases after consuming one input event.

Signature Return Key

The only colored element in the bar. Warm amber gradient with subtle glow and inner shine. The "do the thing" button — visually elevated to feel like a satisfying, definitive action.

Escape Isolation

Far left, deliberately separated from modifiers by a divider. Critical for vim :q! moments. Never accidentally hit when reaching for shift or ctrl.

Space Economy

By moving arrows to the joystick gesture, we freed ~80px of horizontal space. That space now holds four system modifiers that were previously impossible to fit. No more missing Ctrl+C.
