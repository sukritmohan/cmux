cmux Mobile: Pane Type Switcher

The surface strip evolves into a context-aware tab bar. A pane type dropdown at the top-right corner lets users switch between Terminal, Browser, File Explorer, and Shell views. Each pane type brings its own tab bar and bottom controls. The workspace drawer remains workspace-only.

Updated Interaction Model

The pane type dropdown replaces the old Browser/Files/Shell quick actions from the workspace drawer. Pane switching is now a first-class top-level action, always one tap away.

Top Bar Anatomy

The top bar is split into two zones. Left: a scrollable tab bar showing tabs for the current pane type (terminal surfaces, browser tabs, etc.). Right: the pane type dropdown trigger, separated by a subtle vertical divider. The dropdown always shows the current pane type name and a chevron.

Tab Bar Adapts Per Pane Type

Terminal and Browser show scrollable tabs with a "+" button. File Explorer replaces the tab bar with a breadcrumb path. Shell (Mosh) shows session tabs. The tab bar region is the flexible element; the dropdown trigger is pinned right.

Bottom Bar Adapts Too

Terminal shows the modifier key bar (Esc, Ctrl, Alt, Cmd, Tab, arrows). Browser shows navigation controls (back, forward, URL pill, share). File Explorer shows file actions (new file, new folder, sort). Each pane type has purpose-built controls.

Transition Animations

How the pane type switch feels. The goal is a fast, seamless swap that preserves spatial context.

Dropdown Open

Scrim fades in (opacity 0 to 0.35, 150ms ease-out). Menu scales from 0.95 to 1.0 with spring easing (200ms, overshoot 1.08). Origin point: top-right corner of the menu. Items stagger in with 30ms delay each (opacity + 4px translateY).

Pane Type Switch

Outgoing content fades out + slides left 12px (150ms). Incoming content fades in + slides from right 12px (200ms, 50ms delay). Tab bar cross-fades in place (150ms). Bottom bar swaps with a subtle 4px upward slide (200ms). Total perceived transition: under 300ms.

Dropdown Close (No Selection)

Menu scales to 0.97 + fades out (120ms ease-in). Scrim fades out (150ms). Faster than open for responsive feel.
